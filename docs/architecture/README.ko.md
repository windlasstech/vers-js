# 아키텍처 사양

<div align="center">

[English](README.md) | 한국어

</div>

이 디렉터리에는 `vers-js` v0.1.0의 구현 사양이 있습니다. 이 문서들은 수락된 ADR들을 구현 및 테스트 코드가 따라야 할 구체적인 계약으로 변환한 것입니다.

## 개발 흐름

이 프로젝트는 **SDD(Spec-Driven Development, 사양 주도 개발)** 방법론을 따릅니다.

1. **의사결정 맥락**: ADR들을 통해 특정 아키텍처가 선택된 *이유*를 이해합니다.
2. **기술 명세**: 아키텍처 사양을 통해 *정확한 관측 가능한 동작*을 정의합니다.
3. **구현**: 사양에 따라 라이브러리와 테스트를 구현합니다.

## 사양 작성 순서

새 사양을 작성하거나 기존 사양을 검토할 때 다음 순서를 따르세요.

1. `scope-and-invariants.md`: 먼저 경계를 설정합니다.
2. `public-api.md`: 호출 가능한 표면을 정의합니다.
3. `data-model-and-canonical-output.md`: 성공 값을 정의합니다.
4. `character-encoding.md`: 입력 처리 규칙을 정의합니다.
5. `parser-phases.md`: 실행 순서를 정의합니다.
6. `diagnostics.md`: 실패 값을 정의합니다.
7. `fixtures.md`: 테스트 기대치를 정의합니다.
8. `resource-limits.md`: 리소스 경계를 정의합니다.
9. `build-and-test.md`: 스캐폴드 및 검증을 정의합니다.
10. 필요 시 신규 문서를 추가 작성합니다.

## v0.1.0 범위

**범위 내:**

- 정규화한 VERS 구문 검증
- 파싱한 선언문 메타데이터 (`VersRange`, `VersConstraint`)
- 정규 문자열 투영
- 구문 전용 타입 검증
- 단일 패스 퍼센트 디코딩
- 원본 입력 기반 영향 범위를 포함한 진단 정보
- 공식 파싱/정규 적합성 픽스처

**범위 밖:**

- 버전 비교 또는 포함 관계
- 생태계별 네이티브 범위 표현 변환
- 의미론적 정렬 또는 단순화
- 알려진 타입 레지스트리 적용
- 경고, 수정, 강제 변환 모드
- 취약점 해석 또는 VEX 의미론
